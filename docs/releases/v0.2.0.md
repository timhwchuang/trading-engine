pip install "trading-engine[shioaji] @ git+https://github.com/timhwchuang/trading-engine.git@v0.2.0"

## Highlights

This release implements the CodeReview#2 improvement plan: documentation for live failure modes, read-only state observation, signal validation guards, and CI quality checks. Re-review ([CodeReview#2-re.md](../../CodeReview#2-re.md)) confirms readiness for **personal UAT** (simulation → paper → small live).

### Before you go live

1. Read [docs/LIVE_SAFETY.md](../LIVE_SAFETY.md) end-to-end
2. Complete [docs/UAT_CHECKLIST.md](../UAT_CHECKLIST.md) in your consuming app
3. Use **`engine.get_state_snapshot()` only** — never assign to `engine.position_qty`, `engine.is_pending`, etc.
4. Implement `AlertPort` so **CRITICAL** alerts (disconnect, pending timeout, relogin exhaustion) reach you

### New API

```python
snap = engine.get_state_snapshot()  # frozen, read-only
```

Invalid strategy signals (`qty <= 0`, bad intent, entry while pending, etc.) are rejected with warnings — see `tests/runtime/test_signal_validation.py`.

### Breaking / behavior changes (0.x)

- Strategies that returned malformed `OrderSignal` may now see kernel warnings and no arm (previously could arm invalid state). Fix strategy output.
- `setup_async_logging()` defaults to configuring root logger; use `configure_root=False` if your app already configured logging (see README).
